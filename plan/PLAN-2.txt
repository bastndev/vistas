# ViewPulse

A lightweight, framework-agnostic page view tracking library focused on one thing: **accurate page view counting**.

Unlike full analytics platforms, ViewPulse is designed to be small, fast, privacy-friendly, and easy to integrate into any JavaScript framework.

---

# Goals

- 🚀 Extremely lightweight (< 3 KB gzip target)
- ⚡ Zero runtime dependencies (client)
- 🔒 Privacy-friendly
- 🧩 Framework agnostic
- 📦 Bring Your Own Database (BYOD)
- 🎨 No built-in UI
- 🛡 Smart deduplication
- 🔌 Adapter-based architecture
- 📈 Accurate page view counting

---

# Core Features

## Smart Deduplication

Prevent duplicate page view increments caused by:

- Page refreshes
- Accidental reloads
- Browser back/forward navigation

Uses configurable browser storage:

- localStorage
- sessionStorage
- AsyncStorage (React Native)
- Custom Storage Adapter

Example:

```ts
cooldown: "24h"
```

If the page has already been counted during the cooldown period:

- ❌ Don't increment
- ✅ Fetch the current count only

---

## Framework Agnostic

Works everywhere.

### Supported

- Vanilla JavaScript
- Astro
- React
- Next.js
- Vue
- Nuxt
- Svelte
- SvelteKit
- React Native

## Bring Your Own Database (BYOD)

ViewPulse does not force a database.

Choose whatever you prefer.

Official adapters:

- Redis
- Upstash Redis
- Supabase
- Turso
- SQLite
- PostgreSQL

Future adapters can be published independently.

---

## Real-Time Counts

Every request returns the latest value from the database.

Example:

```ts
const views = await tracker.track("/blog/my-post");

console.log(views);
// 15872
```

---

## Tiny Client

Target size:

```
< 3 KB gzip
```

No unnecessary runtime dependencies.

---

## Zero UI

ViewPulse only returns data.

You are free to build your own UI.

Example:

```astro

```

or

```tsx

```

# Client Flow

```
User opens page
        │
        ▼
Check local cache
        │
        ├──────────────┐
        │              │
expired             valid
        │              │
        ▼              ▼
POST /views        GET /views
        │              │
Increment        Fetch count
        │              │
        └──────┬───────┘
               ▼
Return latest count
```

---

# Server Flow

```
Request
   │
   ▼
Validate
   │
   ▼
Bot Filter
   │
   ▼
Increment Database
   │
   ▼
Return Updated Count
```

# Default API

```ts
const tracker = createTracker({
    endpoint: "/api/views",
    cooldown: "24h"
});

const views = await tracker.track("/lynx-theme-pro");
```

---

# Vanilla API

```ts
import { trackView } from "@viewpulse/core";

const views = await trackView("/blog");
```

# Server API

## Create Adapter

```ts
import { redisAdapter } from "@viewpulse/server/redis";

const db = redisAdapter({
    url,
    token
});
```

---

## Increment

```ts
await db.increment("/blog");
```

---

## Get Views

```ts
await db.get("/blog");
```

---

## Get Multiple

```ts
await db.getMany([
    "/",
    "/blog",
    "/docs"
]);
```

# Philosophy

ViewPulse is **not** an analytics platform.

It does **one thing**:

> Count page views accurately, efficiently, and with complete control over your data.

Everything else is left to the developer.
